# 🎯 项目根本目的

> **通过盘中异常监控、收盘数据追踪、全球市场简报、全市场基金推荐，辅助用户及时止盈止损、发现更好标的、了解市场环境，最终服务于"买卖基金赚钱"这个根本目的。**

**项目做得好 ≠ 赚钱。** 所有改动的落脚点应该是让用户更有可能赚钱，而不是让代码更漂亮。

## 代码规范

**为什么要在意：** 你要看代码，代码乱了你看着烦，改起来也容易出Bug。Bug导致数据错→亏钱。

- **类型注解**：新代码必须加，已有的保持不动
- **阈值**：动态计算（历史百分位），不硬编码
- **复杂度**：一个函数超过100行就要考虑拆了
- **命名**：函数名要让人一眼看出干什么，不要用 `_cfg` `_pe` 这种缩写
- **注释**：复杂逻辑写注释，简单的不写（代码本身要能读）
- **提交**：每个commit干一件事，commit message 写清楚干了什么

## 质量标准

- 修Bug/改代码后：mypy 零错误 + pytest 通过
- 核心逻辑（净值计算/推送/阈值）要有测试覆盖
- 不要引入AI扮演的"专家"来背书
