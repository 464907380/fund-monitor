# 评分维度预设功能定义

## 概述
允许用户将当前评分维度配置（启用状态、权重、评分曲线）保存为命名预设，后续可一键切换不同预设，无需手动逐项调整。

## 用户故事

**故事 1：快速切换策略**
> 作为基金投资者，我在不同的市场环境下使用不同的评分策略。牛市时更看重收益类维度，熊市时更看重回撤和波动率等风控维度。有了预设功能，我可以在「进攻型」和「防守型」预设之间一键切换。

**故事 2：策略分享与对比**
> 作为用户，我想把精心调参的评分方案保存下来，以后可以快速复现，或与自己的历史方案做对比。

**故事 3：临时调整不丢失**
> 我在系统预设的基础上临时调整了几个维度的权重，想先试试效果又不希望覆盖原来的预设。我可以另存为一个新预设，原有预设保持不变。

## 功能需求

### FR-1：预设管理入口

在评分权重卡片中新增「预设」操作区，位于维度表格上方：

```
┌─ 评分权重 ─────────────────────────────┐
│ ┌─ 预设 ──────────────────────────┐    │
│ │ [预设下拉框 ▾]  [加载] [保存] [删除] │   │
│ │ [另存为...] 输入框                │   │
│ └──────────────────────────────────┘    │
│                                         │
│ [✔] 近3月收益   权重: [ 10]%  📐 ✖      │
│ [✔] 近1月收益   权重: [ 13]%  📐 ✖      │
│ ...                                     │
│ 合计: 100%          [💾 保存权重]       │
└─────────────────────────────────────────┘
```

### FR-2：预设数据模型

预设保存以下维度配置的完整快照（**含评分曲线**）：

```json
{
  "scoring_presets": {
    "系统默认": {
      "dims": [
        {"name": "近3月收益", "key": "m3", "weight": 0.12, "enabled": true, "curve": {"points": [[0,0],[30,50],[60,80],[90,100]]}, ...},
        ...
      ]
    },
    "进攻型": {
      "dims": [
        {"name": "近1年收益", "key": "y1", "weight": 0.20, "enabled": true, "curve": {"points": [[0,0],[50,50],[100,80],[150,100]]}},
        {"name": "夏普比率", "key": "sharpe", "weight": 0.15, "enabled": true, ...},
        ...
      ]
    },
    "防守型": {
      "dims": [
        {"name": "最大回撤", "key": "max_dd", "weight": 0.25, "enabled": true, ...},
        {"name": "波动率", "key": "volatility", "weight": 0.15, "enabled": true, ...},
        ...
      ]
    }
  }
}
```

> **设计决策**：预设保存**完整快照**，包含每个维度的权重、启用状态和评分曲线断点。加载预设时，维度表格的权重值、复选框、曲线编辑器的所有断点全部替换为预设保存时的状态。这样切换预设后评分行为完全一致，实现"一键完整还原评分策略"的目标。

存储位置：`config.json` 的 `scoring_presets` 字段（与 `scoring.dims` 并列）。

### FR-3：预设操作

#### 加载预设（FR-3.1）
- 从下拉框选择预设名称，点击「加载」
- 确认对话框：「加载预设将替换当前所有维度配置，确定继续？」
- 确认后：维度表格所有行替换为预设的快照数据
- 权重合计、评分曲线、启用状态全部恢复为预设保存时的状态
- 页面标记为「未保存」（保存按钮恢复可点击状态）
- 预设「系统默认」始终存在，不可删除

#### 保存预设（FR-3.2）
- 点击「保存」：用当前维度表格的**完整状态**覆盖当前选中的预设
- 点击「另存为」：弹出输入框，输入新预设名称，保存当前配置为新预设
- 预设名称不能为空，不能与已有预设重名
- 保存成功后下拉框自动选中该预设

#### 删除预设（FR-3.3）
- 点击「删除」：确认后删除选中的预设
- 「系统默认」预设不可删除

### FR-4：预设与当前维度的关系

- 预设仅存储在 `config.json`，不改变 `scoring.dims` 的数据结构
- 点击「加载」预设时，将预设的 dims 写入 `scoring.dims`，但**不自动保存到文件**
- 用户仍需点击「💾 保存权重」将加载的预设持久化
- 页面加载时，`scoring.dims` 的当前值不受预设影响

### FR-5：预设数据持久化

- 预设数据保存在 `config.json` 的 `scoring_presets` 字段
- 单独通过 `POST /api/dims-presets` 接口读写，不与 `POST /api/dims` 耦合

#### API 接口

**GET /api/dims-presets**
返回所有预设列表：
```json
{
  "ok": true,
  "presets": {
    "系统默认": { "dims": [...] },
    "进攻型": { "dims": [...] },
    "防守型": { "dims": [...] }
  },
  "current": "系统默认"
}
```

**POST /api/dims-presets**
保存/删除预设：
```json
// 保存
{"action": "save", "name": "进攻型", "dims": [...]}
// 另存为
{"action": "save_as", "name": "新预设名", "dims": [...]}
// 删除
{"action": "delete", "name": "进攻型"}
```

响应：
```json
{"ok": true, "presets": {...}, "current": "进攻型"}
```

### FR-6：内置预设

系统应预置至少 3 个预设供用户选择：

| 预设名 | 特点 | 侧重维度 |
|--------|------|---------|
| **系统默认** | 均衡型，覆盖收益+风控+质量 | 当前出厂配置的 20 维均等分布 |
| **进攻型** | 高收益导向，适合牛市 | y1(20%), m3(15%), sharpe(12%), profit_ratio(10%), annual_return(10%) |
| **防守型** | 风险控制优先，适合熊市 | max_dd(20%), volatility(15%), calmar(12%), max_loss_days(10%), sortino(10%) |
| **短炒型** | 短期动量捕捉 | f5(25%), m1(20%), m3(15%), win_rate(10%), recovery(8%) |

## 非功能需求

| 需求 | 说明 |
|------|------|
| 即时生效 | 加载预设后立即更新界面，无需刷新页面 |
| 数据完整性 | 预设保存失败时不影响当前正在编辑的维度配置 |
| 预设数量上限 | 最多 20 个预设（含内置），超出时提示 |
| 向后兼容 | 无 `scoring_presets` 字段的旧版 `config.json` 正常使用，仅无预设功能 |

## 不在此功能范围内

- ❌ 预设的导入/导出（跨设备同步）
- ❌ 预设的自动切换（根据市场条件）
- ❌ 预设的热门度/评分功能
