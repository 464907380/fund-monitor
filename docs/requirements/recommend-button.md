# 「▶ 运行推荐」按钮功能定义

## 触发方式
用户在管理页面点击「▶ 运行推荐」按钮

## 前置条件：可配置筛选参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| 拉取数量 | 从全市场排行 API 拉取多少只基金作为候选池 | 200 |
| 最低近 1 年收益 | 近 1 年收益低于此值的基金直接剔除 | 20% |
| 展示条数 | 最终榜单展示多少只 | 20 |

## 功能流程

### Step 1 — 保存筛选规则
将用户当前设置的三个参数持久化到 `config.json`，确保后续所有流程使用同一套规则。

### Step 2 — 启动后台推荐任务
启动一个异步子进程执行全市场基金扫描：

1. 调用天天基金排行 API，按「拉取数量」获取全市场排行
2. 按「最低近 1 年收益」剔除不合格基金
3. 对剩余基金逐只拉取净值数据，按当前「评分维度」配置计算 20+ 维评分
4. 按综合得分排序，保存结果到文件

> 此步骤约耗时 4~16 分钟，不可阻塞用户操作。

### Step 3 — 进度反馈
- 按钮变为「⏳ 推荐中 x%」，实时显示当前进度
- 进度条同步更新
- 用户可正常操作页面其他功能

### Step 4 — 推荐完成，自动刷新两张数据表并恢复按钮

推荐计算完成后，按钮保持禁用，依次刷新两张表并实时显示进度：

1. **表一：市场优选排行（全维度）**
   - 状态提示：「⏳ 刷新表格 (1/2)...」
   - 按钮文字：「⏳ 刷新表格 (1/2)」
   - 响应快（约 1 秒），因数据来自本地缓存文件

2. **表二：自选基金完整数据**
   - 状态提示：「⏳ 刷新表格 (2/2)...」
   - 按钮文字：「⏳ 刷新表格 (2/2)」
   - 耗时较长（约 30 秒~3 分钟），因需要逐个拉取实时净值数据

3. **两表刷新完毕，按钮恢复：**
   - ✅ 按钮文字 →「▶ 运行推荐」
   - ✅ 按钮恢复可点击
   - ✅ 状态提示 →「✔ 刷新完成」（4 秒后自动消失）
   - ✅ 进度条 → 保持 100% 后逐渐归零

> **关键体验原则**：按钮在整个推荐→刷新过程中始终禁用，避免用户误操作重复启动。

## 非功能需求

| 需求 | 说明 |
|------|------|
| 幂等性 | 如果推荐任务已在运行，第二次点击提示「推荐任务正在运行中」，不重复启动 |
| 配置即时生效 | 点击前修改的筛选参数和评分维度权重，必须在本轮推荐中生效 |
| 刷新友好 | 用户刷新页面后，已完成的推荐数据应该恢复显示 |
| 不触发通知 | 推荐任务**不发送**企业微信、邮件等任何推送通知 |

## 不在此功能范围内

- ❌ 生成晚报 HTML（属于「📊 生成晚报」按钮）
- ❌ 发送企业微信/邮件推送（同上）
- ❌ 盘中实时监控（属于定时任务 `fund_monitor.py`）
- ❌ 全球股市简报（属于定时任务 `global_briefing.py`）
